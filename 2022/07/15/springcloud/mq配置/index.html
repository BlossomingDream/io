消息队列Rabbit-MQ的配置

  • publisher：生产者
  • consumer：消费者
  • exchange：交换机，负责消息路由
  • queue：队列，存储消息
  • virtualHost：虚拟主机，隔离不同租户的exchange、queue、消息的隔离

SpringAMQP

  • https://spring.io/projects/spring-amqp
  • 自动声明队列、交换机及其绑定关系
  • 基于注解的监听器模式，异步接收消息
  • 封装了RabbitTemplate工具，用于发送消息

1.引入amqp依赖

<!--AMQP依赖，包含RabbitMQ-->
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-amqp</artifactId>
</dependency>

2.yml配置amqp

生产者消费者都需要进行配置

# 五种消息模式
- Publisher：生产者，也就是要发送消息的程序，但是不再发送到队列中，而是发给X（交换机）
- Exchange：交换机，图中的X。一方面，接收生产者发送的消息。另一方面，知道如何处理消息，例如递交给某个特别队列、递交给所有队列、或是将消息丢弃。到底如何操作，取决于Exchange的类型。Exchange有以下3种类型：
  - Fanout：广播，将消息交给所有绑定到交换机的队列
  - Direct：定向，把消息交给符合指定routing key 的队列
  - Topic：通配符，把消息交给符合routing pattern（路由模式） 的队列
- Consumer：消费者，与以前一样，订阅队列，没有变化
- Queue：消息队列也与以前一样，接收消息、缓存消息。


- Topic交换机接收的消息RoutingKey必须是多个单词，以 `**.**` 分割
- Topic交换机与队列绑定时的bindingKey可以指定通配符
- `#`：代表1个或多个词
- `*`：代表1个词



# 基于注解声明队列和交换机
```java

/*
- @Binding：绑定关系
- @Queue：声明队列
- @Exchange：交换机
- 属性key：routingkey
*/

//============direct模式=====================
@RabbitListener(bindings = {
    @QueueBinding(
            value = @Queue(name = MqConstant.DIRECT_QUEUE1),
            exchange = @Exchange(name = MqConstant.DIRECT_EXCHANGE, type = ExchangeTypes.DIRECT),
            key = {MqConstant.ROUTING_BLUE, MqConstant.ROUTING_RED}
    )
})
public void listenDirectQueue1(String message) throws InterruptedException {
    log.warn("listenDirectQueue1 --> message:{}", message);
}

  //topic-----------------
    @RabbitListener(bindings = {
            @QueueBinding(
                    value = @Queue(name = MqConstant.TOPIC_QUEUE1),
                    exchange = @Exchange(name = MqConstant.TOPIC_EXCHANGE, type = ExchangeTypes.TOPIC),
                    key = {MqConstant.ROUTING_PREFIX_CHINA}
            )
    })
    public void listenTopicQueue1(String message) throws InterruptedException {
        log.info("listenTopicQueue1 --> message:{}", message);
    }

3.消息序列化

  • JDK序列化方式并不合适。我们希望消息体的体积更小、可读性更高，因此可以使用JSON方式来做序列化和反序列化。

1.引入依赖

  • 在publisher和consumer两个服务都要进行配置
    <dependency>
    <groupId>com.fasterxml.jackson.dataformat</groupId>
    <artifactId>jackson-dataformat-xml</artifactId>
    <version>2.9.10</version>
</dependency>

2.配置消息转换器

/**
 * 消息转换器配置类
 *
 * @Name MessageConverterConfig
 * @Author admin
 * @Date 2022-08-03 17:50
 */
@Configuration
public class MessageConverterConfig {

    @Bean
    public MessageConverter messageConverter(){
        return new Jackson2JsonMessageConverter();
    }

}
